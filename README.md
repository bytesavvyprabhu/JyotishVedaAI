# JyotishVedaAI

A RAG-based Vedic astrology assistant. Give it a name, place/date/time of
birth, and a question — it computes a full sidereal birth chart (Swiss
Ephemeris + Lahiri ayanamsa) and answers using classical astrology rules
retrieved for that chart, generated by an LLM (Groq/Llama).

Two parts:
- **Backend** (this folder) — FastAPI service that geocodes the birth place,
  computes the chart, retrieves relevant classical rules, and generates the
  answer.
- **Mobile app** ([`app/`](app/)) — Flutter app (Android/iOS) with a birth
  details form and a results screen.

## How it works

1. **Geocode** ([Geocode.py](Geocode.py)) — resolves the birth place to
   lat/lon (Nominatim/OpenStreetMap) and the UTC offset at that place and time.
2. **Chart computation** ([chart.py](chart.py)) — computes the ascendant,
   sidereal positions of the 9 grahas (planets + Rahu/Ketu), their houses,
   nakshatra/pada, Moon sign, and the Vimshottari Mahadasha sequence.
3. **Retrieval** ([ask.py](ask.py)) — classifies the question's topic
   (marriage, career, wealth, health, personality, remedy), then builds
   chart-aware queries (e.g. *"Saturn in the 7th house in Capricorn"*) instead
   of just embedding the raw question, and retrieves the closest classical
   rules from a Chroma vector store.
4. **Generation** ([ask.py](ask.py)) — augments a prompt with the chart facts
   and retrieved rules, and calls Groq's Llama 3.3 70B to generate a warm,
   specific reading.
5. **Ingestion** ([Ingest.py](Ingest.py)) — loads classical rules from
   `data/*.json` and any `data/*.pdf` reference texts, chunks/embeds them
   (via `sentence-transformers`), and upserts them into the local Chroma
   vector store (`chroma_db/`).

## Project structure

```
JyotishVedaAI/
├── main.py          # FastAPI app, POST /consult endpoint
├── Geocode.py        # Place -> lat/lon/timezone offset
├── chart.py          # Birth chart computation (Swiss Ephemeris)
├── ask.py            # RAG retrieval + LLM answer generation
├── Ingest.py          # Loads data/*.json + data/*.pdf into the vector store
├── data/              # Classical astrology rules (JSON) and reference PDFs
├── chroma_db/          # Generated vector store (gitignored, rebuild with Ingest.py)
├── requirements.txt
├── .env.example
└── app/                # Flutter mobile app (see app/README.md)
```

## Backend setup

Requires Python 3.10+.

```bash
python -m venv .venv
.venv\Scripts\activate        # Windows
# source .venv/bin/activate   # macOS/Linux

pip install -r requirements.txt
```

Copy `.env.example` to `.env` and add a free Groq API key
([console.groq.com](https://console.groq.com)):

```
GROQ_API_KEY=your_key_here
```

Build the vector store from the rules in `data/`:

```bash
python Ingest.py
```

Run the API:

```bash
uvicorn main:app --reload
```

To let a phone on the same Wi-Fi (or the Flutter app) reach it, bind to all
interfaces instead:

```bash
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

### API

**POST `/consult`**

```json
{
  "name": "Test User",
  "place_of_birth": "Varanasi, Uttar Pradesh, India",
  "date_of_birth": "1995-08-15",
  "time_of_birth": "10:30",
  "question": "When will I get married?"
}
```

Returns the computed chart plus a generated `answer` string. Interactive docs
are available at `http://localhost:8000/docs` once the server is running.

## Mobile app

The Flutter app in [`app/`](app/) provides a native UI for the API above —
birth details form, chart summary, mahadasha timeline, and the AI reading.
See [app/README.md](app/README.md) for how to run it against this backend
(emulator, simulator, or a physical phone over Wi-Fi).
